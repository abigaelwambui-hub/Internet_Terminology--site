Architecture

Client–Server Architecture
Overview
The client–server model is the foundation of most internet applications. In this setup, the client (like a web browser or mobile app) requests services or resources, while the server provides them.
Key Features
- Separation of roles: Clients handle user interaction, servers handle data and services.
- Scalability: Servers can serve multiple clients simultaneously.
- Security: Sensitive data is stored and managed on the server side.
Example
When you visit a website, your browser (client) sends a request to the server. The server responds with the webpage files, which the browser displays.
